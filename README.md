# Telegram Bridge Core (Proxy Gateway)

**Telegram Bridge Core** — локальный прокси-клиент с графическим интерфейсом на базе WPF (.NET 8.0). Приложение предназначено для маршрутизации трафика и создания прокси-шлюза (в том числе с использованием Cloudflare Tunnel). 

Оно создает локальный WebSocket-сервер, который проксирует подключения к выбранным серверам-ретрансляторам (Datacenters). Встроенная утилита `cloudflared` позволяет также открывать этот локальный порт наружу.

## 💻 Требования к системе

Поскольку это приложение с графическим интерфейсом на базе **Windows Presentation Foundation (WPF)**, оно работает **только на операционной системе Windows**.

Для запуска и сборки вам потребуется:
- Установленный .NET 8 SDK (для разработки/сборки из исходников).
- Операционная система Windows (10/11).

> **Примечание:** Если вы хотите передать это приложение другому человеку или использовать его без установки SDK, вам нужно будет собрать его как **единый исполняемый файл (.exe)**. Инструкция приведена ниже.

---

## 🚀 Запуск из исходного кода (для разработчика)

Если вы находитесь в директории проекта или работаете через редактор кода (например, Visual Studio или Rider), для запуска приложения можно использовать консольную команду:

```bash
dotnet run
```
Также вы можете просто открыть файл `TelegramProxy.csproj` в Visual Studio 2022 и нажать кнопку **Start (F5)**.

---

## 📦 Как сделать EXE-файл (Сборка готового приложения)

Чтобы превратить проект в готовое приложение в виде одного файла **.exe** (которое можно просто скопировать на флешку и запускать на других ПК без установки дополнительных программ), выполните следующую команду в папке с вашим проектом (`TelegramProxy.csproj`):

```bash
dotnet publish -c Release -r win-x64 --self-contained -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true
```

**Где найти готовый `.exe` после сборки?**
После успешного завершения команды перейдите в папку:
`bin\Release\net8.0-windows\win-x64\publish\`
Там вы увидите ваш готовый `TelegramProxy.exe`. Это полностью самостоятельное приложение.

## 🛠 Как пользоваться приложением

1. Запустите приложение (через `dotnet run` или готовый `TelegramProxy.exe`).
2. В секции **Endpoint Pipeline Setup** укажите желаемый `Local Port` (локальный порт, например, 8080).
3. Выберите из списка `Target Relay Node` — сервер назначения.
4. По желанию, включите галочку **Route via Cloudflare Tunnel**, если хотите использовать Cloudflare для туннелирования (при первом запуске утилита `cloudflared.exe` будет автоматически извлечена и запущена).
5. Нажмите **Initialize Bridge** для старта.
6. В строке **Connection URL** появится ссылка, к которой нужно подключаться. 
7. Для остановки прокси нажмите **Terminate**. Логи операций и статистика трафика будут отображаться в основном окне.
